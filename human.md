# راهنمای استفاده از Open Notebook

به مخزن Open Notebook خوش آمدید! این پروژه یک جایگزین متن‌باز و متمرکز بر حریم خصوصی برای Google Notebook LM است. این ابزار به شما امکان می‌دهد تا محتوای تحقیقاتی خود را مدیریت کنید، با هوش مصنوعی در مورد آن‌ها گفتگو کنید و حتی پادکست‌های صوتی تولید کنید.

## ویژگی‌ها و امکانات اصلی

این پروژه امکانات گسترده‌ای را در اختیار شما قرار می‌دهد:

*   **حریم خصوصی (Privacy-First):** تمام داده‌های شما به صورت محلی و تحت کنترل شما ذخیره می‌شوند.
*   **پشتیبانی از چندین مدل هوش مصنوعی:** امکان استفاده از 16+ ارائه‌دهنده مختلف از جمله OpenAI, Anthropic, Ollama, Google Gemini و غیره.
*   **تولید پادکست:** قابلیت تولید پادکست‌های حرفه‌ای با چندین گوینده (Speaker) بر اساس محتوای شما.
*   **جستجوی هوشمند:** جستجوی تمام‌متن و برداری (Vector Search) در تمام اسناد و محتواها.
*   **رابط کاربری سه ستونه:** طراحی کارآمد شامل بخش‌های منابع (Sources)، یادداشت‌ها (Notes) و گفتگو (Chat).
*   **پشتیبانی از انواع محتوا:** پشتیبانی از PDF، ویدئو، فایل‌های صوتی، صفحات وب و اسناد متنی.
*   **API کامل:** دسترسی برنامه‌نویسی کامل از طریق REST API.

## ساختار کلی پروژه

این پروژه از بخش‌های زیر تشکیل شده است:

1.  **Frontend:** با استفاده از **Next.js** و **React** ساخته شده است (پوشه `frontend/`). پورت پیش‌فرض: `8502`.
2.  **Backend (API):** با استفاده از **Python** و **FastAPI** توسعه داده شده است (پوشه `api/`). پورت پیش‌فرض: `5055`.
3.  **Database:** از **SurrealDB** برای ذخیره‌سازی داده‌ها استفاده می‌کند. پورت پیش‌فرض: `8000`.
4.  **Worker:** پردازش‌های پس‌زمینه (مانند تولید پادکست) را انجام می‌دهد.

## نحوه نصب و راه‌اندازی

### استفاده از Docker (روش پیشنهادی)

ساده‌ترین راه برای اجرای برنامه استفاده از Docker Compose است. دستورالعمل‌های دقیق در فایل `README.md` موجود است، اما به طور خلاصه:

1.  فایل `docker-compose.yml` را ایجاد یا استفاده کنید.
2.  دستور `docker compose up -d` را اجرا کنید.
3.  برنامه در آدرس `http://localhost:8502` در دسترس خواهد بود.

**نکته مهم:** پورت‌های `8502` (فرانت‌اند) و `5055` (API) هر دو باید در دسترس باشند.

### راه‌اندازی برای توسعه‌دهندگان (Local Development)

اگر می‌خواهید کد را تغییر دهید یا توسعه دهید:

1.  **پیش‌نیازها:** مطمئن شوید که `uv` (برای پایتون)، `Node.js` و `Docker` نصب هستند.
2.  **نصب وابستگی‌ها:**
    *   در ریشه پروژه: `uv sync`
    *   در پوشه `frontend`: دستور `npm install`
3.  **اجرای تمام سرویس‌ها:**
    از دستور زیر در ریشه پروژه استفاده کنید تا دیتابیس، API، ورکر و فرانت‌اند همزمان اجرا شوند:
    ```bash
    make start-all
    ```

## دستورات مفید (Makefiles)

در این مخزن از `Makefile` برای ساده‌سازی دستورات استفاده شده است:

*   `make start-all`: اجرای تمام سرویس‌ها.
*   `make stop-all`: متوقف کردن تمام سرویس‌ها.
*   `make run` یا `make frontend`: اجرای فرانت‌اند به تنهایی.
*   `make api`: اجرای سرور API.
*   `make worker`: اجرای ورکر پس‌زمینه.
*   `make lint`: بررسی تایپ‌ها با `mypy`.
*   `make ruff`: فرمت‌دهی و بررسی کد با `ruff`.
*   `make database`: اجرای دیتابیس SurrealDB به تنهایی با داکر.

## مستندات بیشتر

*   برای جزئیات دقیق API به `http://localhost:5055/docs` (پس از اجرا) مراجعه کنید.
*   فایل‌های راهنما در پوشه `docs/` موجود هستند.
*   برای قوانین مشارکت در پروژه، فایل `CONTRIBUTING.md` را مطالعه کنید.
